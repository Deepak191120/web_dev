Overview

static web pages are often HTML documents stored as files in the file system and made available by the web server over HTTP(neverthless URLs ending with ".html" are not always static).However,losse interpretations of the term could include web pages stored in a database,and could even include pages formatted using a template and server through an application server,as long as the page served is unchanging and presented essentially as stored.
Static web pages are suitable for content that never or rarely needs to be updated,through modern web template systems are changing this.Maintaining large number of static pages as files can be impractical without automated tools,such as static site generators.Any personalization or interactivity has to run client side,which is restriicting.

  Advantages of a static website

  • provide improved security over dynamics webites(dynamic websites are at risk to web shell attacks if a vulnerability is present).
  • Improved performance for end users compared to dynamic websites.
  • fewer or no dependencies on system such as databases or other application servers.
  • Cost saving from utilizing cloud storage,as opposed to a hosted environment.
  • Security configurations are easy to setup,which makes it more secure.
